確率変数

確率変数とは関数である

確率変数 (random variable) とは 標本点を数値に結びつける関数 である．

たとえばサイコロを振って「\(1\) の目」が出たとしよう．「\(1\) の目が出る」というのは 標本点 のひとつであって，数値「1」そのものではない．この標本点に数値「1」を割り当てるのが確率変数である※．同じように，コイントスの「表 \(\to 0\)，裏 \(\to 1\)」も，アンケートの「男 \(\to 0\)，女 \(\to 1\)」も，どれも観察結果を数値に結びつけている．すなわち確率変数をひとつ定めている例だ．

つまり，確率変数の値とは，標本空間からランダムに選ばれた標本点に結びつけられた値のことである．

「サイコロの 1」という標本点に対して，数値「2」を割り当てても別に構わない．混乱するので普通はそんなことしないが．

大文字と小文字の使い分けについて

ここではクラスからランダムに選び出した学生のテストの得点を確率変数 \(X\) としよう．確率変数は大文字で表記するのが慣例で，小文字 \(x\) は具体的な測定値（たとえば \(70\)※）を表す．「容れ物」が \(X\)，「中身として今回入ってきた値」が \(x\)，という使い分けと思えばよい．小文字 \(x\) のことを，特に 実現値 (realization) と呼ぶこともある．

ここでの単位は「得点」だが，あくまでも単なる数値としての「70」であることに注意．

\(X\) は「（ランダムに選ばれた）人 \(\to\) 得点」という対応を表す関数であり，\(x\) はその出力値にあたる．したがって \(P(X = 70)\) は「規則 \(X\) を働かせた結果がたまたま \(70\) になる確率」を意味する．ノート「標本空間，事象，確率の初歩」で事象 \(A\) の確率を \(P(A)\) と書いたのと同様，ここでは \(X=70\) という 条件式が事象を表している．

確率変数が文脈上明らかな場合は「\(X=\)」を省略し，小文字で \(p(70)\) や \(p(x)\) と略すこともある．意味は \(P(X = 70)\) や \(P(X = x)\) と同じ．大文字 \(P\) は事象に，小文字 \(p\) は実現値に対して使うのが慣例である．

別の量（別の科目の得点や出席日数など）に着目したいときは，別の大文字（\(Y, Z, \ldots\)）を割り当てる．同じ試行を繰り返したり複数人を選んだりするときは，添字をつけて \(X_1, \ldots, X_n\) と並べ，対応する実現値は \(x_1, \ldots, x_n\) と書く．

確率分布

いま，確率変数 \(X\) が値 \(x\) をとる確率を

\[ P(X = x) \]

と表記し，\(X\) のとりうるすべての値 \(x\) について確率 \(P(X = x)\) を定めた規則全体を 確率分布 (probability distribution) と呼ぶ．離散型の場合，値 \(x\) に確率を割り当てる関数 \(p_X(x)=P(X=x)\) を，後述する確率密度関数との対比で 確率質量関数 (probability mass function; PMF) と呼ぶ．※

\(p_{\color{#d97a52}X}(x)\) のように，\(p\) に下付きの \(X\) を付ける理由は，複数の確率変数 \(X, Y, Z, \ldots\) を扱うときに，それぞれの確率質量関数を区別するためである．変数名を \(p(x) \)，\(p(y)\)，\(p(z)\) のように使い分けたり，関数名を \(p(x)\)，\(q(x)\)，\(r(x)\) と使い分けて区別がつく文脈では，下付きを省略する場合も多い．

\(X\) がある値 \(a\) と \(b\) の間に入る確率は

\[ P(a \le X \le b) \]

のように表記する．

同様に，たとえば「合格である」という事象を \(A\) とおき，\(A\) を \(X \ge x_{\mathrm{pass}}\)（仮に \(x_{\mathrm{pass}} = 60\)）と定義するなら，ランダムに選んだ学生が合格である確率は

\[ P(A) \quad \text{または} \quad P(X \ge x_{\mathrm{pass}}) \]

のように表記する．

また，確率変数 \(X\) が正規分布 \(\mathcal{N}(\mu, \sigma^2)\) や二項分布 \(\text{Bi}(n, p)\) のような代表的な確率分布に従うことを表したいときには，

\[ \begin{aligned} X &\sim \mathcal{N}(\mu, \sigma^2),\\ X &\sim \text{Bi}(n, p) \end{aligned} \]

のように表記する．たとえば「\(X \sim \mathcal{N}(\mu, \sigma^2)\)」は「\(X\) は正規分布 \(\mathcal{N}(\mu, \sigma^2)\) に従う」と読む．具体的にどんな分布なのかは後の回で出会う．今は記号だけ覚えておけばよい．

確率密度

確率変数 \(X\) が連続型（実数値）の場合，常に

\[ P(X = x) = 0 \]

となる点に注意しよう．\(X\) の値が厳密に実数 \(x\) に一致する確率はゼロだ．奇妙に感じるかもしれないが，連続的につながった実数の世界では，「ちょうど \(x\)」という 1 点には幅がない．確率を面積とみなせば，その面積がゼロになるのは自然だろう．

そこで連続型では 確率密度 (probability density) という考えを用いて起こりやすさの程度を扱う．確率変数 \(X\) についての 確率密度関数 (probability density function; PDF) を \(f_X(x)\) と表記する．密度それ自体は確率ではなく，ある範囲で積分してはじめて確率になる：

\[ P(a \le X \le b) = \int_a^b f_X(x)\, dx. \]

つまり，密度関数のグラフの下の面積が確率に相当する．

ちなみに，\(-\infty\) から \(\infty\) まで積分すればすべての可能性を覆い尽くすので，このときの積分値（全面積）は必ず \(1\) になる：

\[ \int_{-\infty}^{\infty} f_X(x)\, dx = 1. \]

言い換えれば，グラフ全体の下の面積（\(=1\)）に対する \([a, b]\) 区間の面積比が，まさに \(P(a \le X \le b)\) である．ノート「標本空間，事象，確率の初歩」で解説した 全体に対する部分の面積比 のイメージそのままだ．

もうひとつちなみに，確率の最大値は 1 だが，確率密度関数の値は 1 を超えることもある 点に注意しよう．確率密度関数は積分して（面積として計算して）初めて確率になるのであって，関数の値そのものは確率ではない．

累積確率

確率変数 \(X\) がある値 \(x\) 以下になる確率 \(P(X \le x)\) を 累積確率 (cumulative probability) と呼ぶ．これは \(x\) についての関数とみなせて，累積分布関数 (cumulative distribution function; CDF) と呼び，

\[ F_X(x) = P(X \le x) \]

と表記する．これを使えば

\[ P(a \le X \le b) = F_X(b) - F_X(a) \]

のように書ける ── 連続型のとき厳密．離散型では端点の扱いに微妙な差が出るが，ここでは深入りはしない．

離散型の確率変数の場合は

\[ F_X(x) = \sum_{s \le x} P(X = s). \]

連続型の確率変数の場合は

\[ F_X(x) = \int_{-\infty}^{x} f_X(s)\, ds. \]

定義から，

\[ \begin{aligned} \lim_{x \to -\infty} F_X(x) &= 0\\[3pt] \lim_{x \to \infty} F_X(x) &= 1 \end{aligned} \]

である．

連続型では，累積分布関数を微分したものが確率密度関数になる：

\[ f_X(x) = \frac{dF_X(x)}{dx}. \]

「積み上げ」が累積，「増分」が密度である．

累積確率と対をなすのが 補累積確率 (complementary cumulative probability) ── ある値 \(x\) より大きな値をとる確率 \(P(X > x)\) ── で，補累積分布関数 (complementary cumulative distribution function; CCDF)

\[ \bar{F}_X(x) = 1 - F_X(x) = P(X > x) \]

を用いて表す．「左から積み上げた分」が CDF，「右にどれだけ残っているか」が CCDF．

確率分布や確率密度が分かれば，累積確率（や補累積確率）なんて要らないのでは？と思うかもしれない．しかし「\(X\) が \(x\) 以上になる確率」を求めるなら補累積確率を見るのが手っ取り早いし，「区間 \([a, b]\) に入る確率」も \(F_X(b) - F_X(a)\) と簡潔に書ける．問題に応じた表現の使い分けは有用だ．
